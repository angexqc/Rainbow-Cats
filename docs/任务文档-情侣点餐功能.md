### Task 1.4: 统一错误处理工具
**状态**: ⬜ 待开始
**优先级**: P0
**预估工作量**: 0.5 天
**负责人**: 待分配

**任务描述**:
创建统一的错误处理工具类和日志记录工具。

**文件路径**: `miniprogram/utils/`

**errorHandler.js**:
```javascript
/**
 * 统一错误处理工具
 */
module.exports = {
  /**
   * 处理云函数错误
   * @param {Error} err 错误对象
   * @param {String} operation 操作描述
   * @param {Object} context 上下文信息
   */
  handleCloudError(err, operation, context = {}) {
    console.error(`[云函数错误] ${operation}:`, err);
    console.error('[上下文]', context);

    // 根据错误类型返回友好提示
    if (err.errCode === -1) {
      return { success: false, message: '网络异常，请稍后重试' };
    }
    if (err.errCode === -502001) {
      return { success: false, message: '云函数执行失败' };
    }
    return { success: false, message: err.errMsg || '操作失败' };
  },

  /**
   * 显示错误提示
   * @param {String} message 错误信息
   */
  showToast(message) {
    wx.showToast({
      title: message,
      icon: 'none',
      duration: 2000
    });
  }
};
```

**logger.js**:
```javascript
/**
 * 日志记录工具
 */
module.exports = {
  /**
   * 记录操作日志
   * @param {String} action 操作类型
   * @param {Object} data 数据
   */
  log(action, data) {
    const timestamp = new Date().toISOString();
    console.log(`[${timestamp}] [${action}]`, JSON.stringify(data));
  },

  /**
   * 记录错误日志
   * @param {String} action 操作类型
   * @param {Error} err 错误对象
   * @param {Object} data 数据
   */
  error(action, err, data = {}) {
    const timestamp = new Date().toISOString();
    console.error(`[${timestamp}] [ERROR] [${action}]`, err);
    console.error('[数据]', JSON.stringify(data));
  }
};
```

**验收标准**:
- [ ] 工具类创建成功
- [ ] 导出接口正确
- [ ] 代码有完整注释

## 阶段二：情侣关联模块

### Task 2.1: 情侣配对云函数
**状态**: ⬜ 待开始
**优先级**: P0
**预估工作量**: 1.5 天
**负责人**: 待分配
**依赖**: Task 1.1, Task 1.2, Task 1.4

**任务描述**:
创建情侣配对管理云函数，实现生成配对码、绑定、解绑功能。

**云函数路径**: `cloudfunctions/pairManager/`

**功能接口**:

1. `generatePairCode(event)` - 生成配对码
   - 参数: `{ }`
   - 逻辑: 检查是否已绑定，生成6位数字码，保存24小时有效期
   - 返回: `{ pairCode, expireTime }`

2. `bindPair(event)` - 绑定情侣
   - 参数: `{ pairCode }`
   - 逻辑: 验证配对码有效性，建立双向绑定关系
   - 返回: `{ partnerInfo }`

3. `unbindPair(event)` - 解绑情侣
   - 参数: `{ }`
   - 逻辑: 更新关系状态为已解绑
   - 返回: `{ success }`

4. `getPairInfo(event)` - 获取配对信息
   - 参数: `{ }`
   - 逻辑: 查询当前用户的配对关系
   - 返回: `{ partnerOpenId, bindTime, status }`

**代码规范**:
- 每个函数需添加注释说明
- 使用 logger 记录操作日志
- 使用 errorHandler 处理错误
- 所有操作需校验 OpenId

**验收标准**:
- [ ] 云函数部署成功
- [ ] 配对码生成正常
- [ ] 绑定功能正常
- [ ] 解绑功能正常

## 阶段三：菜单管理模块

### Task 3.1: 菜单管理云函数
**状态**: ⬜ 待开始
**优先级**: P0
**预估工作量**: 2 天
**负责人**: 待分配
**依赖**: Task 1.1, Task 1.2, Task 1.4

**任务描述**:
创建菜单管理云函数，实现菜单的增删改查操作。

**云函数路径**: `cloudfunctions/menuManager/`

**功能接口**:

1. `getMenuList(event)` - 获取菜单列表
   - 参数: `{ category, keyword, page, pageSize }`
   - 逻辑: 查询自己和情侣的菜品，支持分页、筛选、搜索
   - 返回: `{ list, total, hasMore }`

2. `getMenuById(event)` - 根据ID获取菜品
   - 参数: `{ _id }`
   - 逻辑: 查询菜品详情
   - 返回: 菜品详情对象

3. `addMenu(event)` - 添加菜品
   - 参数: `{ title, image, desc, credit, category }`
   - 逻辑: 创建新菜品记录
   - 返回: `{ _id }`

4. `updateMenu(event)` - 更新菜品
   - 参数: `{ _id, title, image, desc, credit, category }`
   - 逻辑: 更新菜品信息（只能更新自己创建的）
   - 返回: 更新结果

5. `toggleMenuStatus(event)` - 切换上架状态
   - 参数: `{ _id, available }`
   - 逻辑: 更新菜品上架状态（双方都可以操作）
   - 返回: 更新结果

6. `deleteMenu(event)` - 删除菜品
   - 参数: `{ _id }`
   - 逻辑: 删除菜品（只能删除自己创建的）
   - 返回: 删除结果

**权限校验**:
- 编辑/删除：校验 `_openid == event.userInfo.openId`
- 上架/下架：可由双方操作
- 所有操作记录日志

**验收标准**:
- [ ] 云函数部署成功
- [ ] 列表查询支持分页、筛选、搜索
- [ ] 增删改查功能正常
- [ ] 权限校验正确
- [ ] 日志记录完整

**缓存策略**:

2. 购物车数据缓存（永久）
3. 配对信息缓存（永久）
4. 订单列表不缓存（需实时）

**缓存实现**:
```javascript
// utils/cache.js
module.exports = {
  set(key, data, expireSeconds = 600) {
    const expireTime = Date.now() + expireSeconds * 1000;
    wx.setStorageSync(key, { data, expireTime });
  },
  get(key) {
    const cached = wx.getStorageSync(key);
    if (!cached) return null;
    if (Date.now() > cached.expireTime) {
      wx.removeStorageSync(key);
      return null;
    }
    return cached.data;
  }
};
```

## 风险与注意事项

| 风险项 | 描述 | 应对措施 |
|--------|------|---------|
| 订阅消息模板审核 | 微信订阅消息模板可能审核不通过 | 准备多个模板方案，提前申请 |
| 图片云存储 | 大量图片可能产生存储费用 | 压缩图片，定期清理 |
| 并发问题 | 双方同时操作可能产生冲突 | 使用乐观锁或版本号控制 |
| 权限控制 | 防止越权操作 | 所有云函数校验 OpenId |
| 积分扣除回滚 | 扣积分成功但订单创建失败 | 记录日志，手动回滚 |
| 消息推送失败 | 用户拒绝授权或API限流 | 不影响订单创建，记录日志 |
| 数据一致性 | 网络异常导致数据不一致 | 使用事务或补偿机制 |
| 性能问题 | 大量数据导致加载缓慢 | 分页加载、数据缓存 |
